# Bot 稳定性与日志隐私设计

## 目标

按以下顺序改进 Bot：

1. 图片下载不再阻塞 iLink 消息监听线程。
2. 应用关闭时停止登录、监听和回复任务。
3. 服务端日志不再记录用户内容或敏感令牌。

现有管理页面仍可查看最近消息。本设计不改变 iLink 协议、模型调用方式或 API 配置。

## 图片处理

复用现有的有界回复线程池，不新增媒体线程池。`processImageItem()` 只校验图片元数据并提交一个用户级任务，随后立即返回。该任务按顺序完成下载、记录图片消息、识别和回复。

同一用户继续使用现有用户锁串行处理，保证图片消息完成前，后续文本消息不会越过它。不同用户仍可由两个回复工作线程并行处理。下载失败继续发送现有的文字降级提示；队列已满时继续使用现有繁忙提示。

## 关闭流程

`BotService` 增加关闭状态。关闭开始后：

- 使当前登录会话失效，并拒绝新的登录和回复任务；
- 将登录状态设为未登录，清空登录凭据、二维码和游标；
- 中断登录轮询线程和消息监听线程；
- 回复线程池停止接收新任务；
- 在统一的五秒期限内等待控制线程和已提交回复任务结束；
- 期限届满后强制中断剩余回复任务。

关闭操作必须可以重复调用，不抛出异常，也不能在关闭后重新启动监听。

## 日志隐私

服务端和文件日志不得记录：

- 二维码链接或二维码内容；
- 完整游标、`contextToken`、AES Key 或媒体下载参数；
- 微信消息正文、模型回复正文或语音转写正文；
- 完整天气请求 URL、天气 JSON 或完整天气结果；
- 外部服务错误响应正文。

微信用户 ID 只允许以掩码形式出现在服务端日志。事件类型、处理阶段、耗时、字节数、HTTP 状态和异常类型可以保留。

管理页面的消息列表和运行日志属于受管理令牌保护的操作界面，保持现有功能，不纳入本次服务端日志删减范围。

## 测试与验收

- 阻塞模拟图片下载时，图片入口应快速返回；释放下载后仍完成识别和回复。
- 同一用户后续文本任务必须等待图片下载、识别和回复任务完成。
- 关闭时先等待已开始的回复任务；五秒期限后强制中断。
- 关闭后登录入口和任务提交入口不得启动新工作。
- 登录与监听线程收到关闭信号后退出，凭据、二维码和游标被清空。
- 静态日志回归测试不得发现上述敏感值被传给服务端日志调用。
- 相关测试和完整 `mvn test` 均通过。

## 范围约束

- 不增加新的线程池或持久化组件。
- 不修改管理页面的消息显示能力。
- 不改动与三项目标无关的包名、类名和依赖。
- 保留当前未提交的 `README.md` 修改，不纳入设计规格提交。
