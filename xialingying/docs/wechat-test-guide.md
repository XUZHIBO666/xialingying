# 微信功能测试手册

> 前置条件：IDEA 中启动项目 → 访问 `http://localhost:8080/bot` → 扫码登录 → Bot 在线

---

## 一、文本对话（已有功能）

### 1.1 普通闲聊

| 发送内容 | 预期结果 |
|---------|---------|
| `你好` | LLM 友好回复 |
| `你是谁` | LLM 自我介绍 |
| `今天心情不错` | LLM 共情回复 |

### 1.2 多轮上下文（✨ 新增：持久化记忆）

| 步骤 | 发送内容 | 预期结果 |
|------|---------|---------|
| 1 | `我叫小明，我喜欢吃火锅` | Bot 记住 |
| 2 | `我喜欢吃什么？` | Bot 回答"火锅" |
| 3 | **重启应用** | — |
| 4 | `我叫什么名字？` | Bot 仍记得"小明"（记忆已持久化） |
| 5 | **删除 `data/conversation-memory.json`** | 清除所有记忆 |
| 6 | `我叫什么名字？` | Bot 不记得（记忆已清除） |

### 1.3 多模态上下文（✨ 新增）

| 步骤 | 发送内容 | 预期结果 |
|------|---------|---------|
| 1 | 发送**一张橘猫图片** | Bot 识别："这是一只橘色的猫" |
| 2 | `它是什么颜色？`（不发图片） | Bot 答"橘色"（上下文感知） |
| 3 | 发送**一条语音**"明天会下雨吗" | Bot 语音回复天气 |
| 4 | `那后天呢？`（纯文字追问） | Bot 知道你在问天气（上下文感知） |

> 上下文有效期 5 分钟，超时后追问失效。

## 三、语音功能（已有 + ✨ 优化）

### 3.1 语音 → 语音回复

| 步骤 | 发送内容 | 预期结果 |
|------|---------|---------|
| 1 | 微信语音："今天天气怎么样" | Bot 回复可播放的 MP3 语音文件 |
| 2 | 微信语音："讲个笑话" | Bot 回复语音文件 |

### 3.2 语音降级测试

| 场景 | 如何触发 | 预期结果 |
|------|---------|---------|
| ASR 失败 | 发送听不懂的语音（噪音/外语） | 文字回复："没有听清，请重新发送一段语音。" |
| TTS 失败 | 不配置 `VOICE_TTS_API_KEY` | 降级为文字回复 |
| 超长语音 | 发送 > 60 秒语音 | 被拒绝，提示重新发送 |

### 3.3 语音 + 文字追问（✨ 新增：多模态上下文）

| 步骤 | 发送内容 | 预期结果 |
|------|---------|---------|
| 1 | 语音："帮我查一下北京的天气" | Bot 语音回复北京天气 |
| 2 | 文字："那明天呢" | Bot 知道你在问"北京明天天气" |

## 四、天气查询

### 4.1 关键词方式（已有）

| 发送内容 | 预期结果 |
|---------|---------|
| `今天杭州天气怎么样` | 返回杭州天气 |
| `北京天气` | 返回北京天气 |
| `Shanghai天气` | 返回上海天气 |

### 4.2 Function Calling 方式（✨ 新增）

| 发送内容 | 预期结果 | 说明 |
|---------|---------|------|
| `今天热不热` | 查询本地天气 | FC 自动判断意图，无需"天气"关键词 |
| `明天会下雨吗` | 查询天气 | FC 自动调用 get_weather 工具 |
| `现在几点了` | 返回当前时间 | FC 自动调用 get_current_time 工具 |
| `今天几号` | 返回日期 | FC 自动调用 |
| `现在什么时候` | 返回时间 | FC 自动调用 |

> 传统关键词路由和 FC 路由同时生效，互不冲突。

## 五、Function Calling 智能路由（✨ 新增）

发送以下消息验证 LLM 自动选择合适的工具：

| 发送内容 | 应调用的工具 | 预期行为 |
|---------|------------|---------|
| `杭州今天多少度` | `get_weather` | 返回温度、天气、湿度等 |
| `深圳会下雨吗` | `get_weather` | 返回降水信息 |
| `几点了` | `get_current_time` | 返回精确时间 |
| `今天星期几` | `get_current_time` | 返回日期 |
| `你好，今天过得怎么样` | 无工具 | 普通闲聊，不调工具 |

> 验证方法：看服务端日志中出现 `[AI] FC回复(含工具)` 说明走了 FC 路径。

## 六、稳定性测试（✨ 新增）

### 6.1 速率限制

| 操作 | 预期结果 |
|------|---------|
| 在 1 秒内连续发送 3 条消息 | 第 3 条收到"你的消息太快了，请稍后再发。" |
| 等待 2 秒后再发 | 正常回复 |

### 6.2 健康检查

| 操作 | 如何访问 |
|------|---------|
| 检查 Bot 是否活着 | `curl http://localhost:8080/bot/health/live` |
| 检查是否就绪（含 iLink 状态） | `curl http://localhost:8080/bot/health/ready` |
| 查看运行指标 | `curl http://localhost:8080/bot/health/metrics`（需带 admin token） |

### 6.3 日志隐私验证

发送任意消息后检查 `logs/` 目录下的日志文件：
- ✅ 不包含完整消息正文
- ✅ 用户 ID 已掩码（如 `user_abc...xyz1`）
- ✅ contextToken 已掩码
- ✅ 天气 JSON 不记录

## 七、快速验证流程（10 分钟）

按顺序执行以下步骤，覆盖所有核心功能：

```
Step 1  文字："你好"                              → 验证基础 LLM 对话
Step 2  文字："我叫小明"                          → 验证记忆功能
Step 3  文字："我叫什么？"                        → 验证记忆（应回答小明）
Step 4  发送一张图片                               → 验证图片识别
Step 5  文字："刚才那张图里有什么？"               → 验证多模态上下文
Step 6  语音："今天天气怎么样"                     → 验证语音→MP3 全链路
Step 7  文字："北京天气"                          → 验证关键词天气
Step 8  文字："明天会下雨吗"                      → 验证 FC 天气
Step 9  文字："画一只小狗"                        → 验证图片生成
Step 10 文字："现在几点了"                        → 验证 FC 时间
Step 11 连续快速发 3 条消息                        → 验证限速
Step 12 curl localhost:8080/bot/health/metrics    → 验证指标端点
```

## 八、注意事项

| 功能 | 所需配置 |
|------|---------|
| LLM 对话 | `AI_API_KEY`（DeepSeek 或其他兼容 API） |
| 图片生成 | `IMAGE_API_KEY`（OpenAI 兼容接口） |
| 图片识别 | `VISION_API_KEY`（SiliconFlow 视觉模型） |
| 语音 ASR | `VOICE_ASR_API_KEY`（SiliconFlow） |
| 语音 TTS | `VOICE_TTS_API_KEY`（SiliconFlow） |
| SILK 解码 | `VOICE_SILK_DECODER_PATH`（本地 decoder 可执行文件） |
| 对话记忆 | 无需额外配置，默认 `./data/conversation-memory.json` |
| FC 工具 | 无需额外配置，自动启用 |
| 健康端点 | 无需额外配置 |

**未配置对应 API Key 的功能会优雅降级**：未配图片生成 → 返回配置提示；未配语音 → 提示重新发送；等。
